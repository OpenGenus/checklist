Getting Started

  • Introduction
    C++ is an extension of the C programming language that is often utilized in the development of system software, applications, and game development.
  • Variables
    C++ supports a variety of data types including: int, char, float, double, bool, and user-defined types.

Control Flow

  • If-Else Statements
    If-Else Statements in C++ are decision making statements executed based on whether a specified condition is met. Code written inside the if statement is executed if the condition is met. Otherwise, the else statement will be run.
  • For Loops
    For Loops are utilized for the repetition of a block of code for a specified number of iterations.
  • Switch Case
    Switch Cases can be used in place of if statements. For each defined case, specified code will be run. If no case is met, the default code will be executed.
  • While Loops
    While Loops perform a specified body of code as long as the control condition remains true.
  • Do While
    Do While differs from that of a while loop in that the loop is guaranteed to be executed at least once before the testing of the condition and will continue as long as the specified condition is met.

Functions

  • User-defined Functions
    User-defined Functions are declared by the user and include a set of statements that are executed given an input. Based on the computation, functions produce output. Function declaration follows the form: return type function name parameter(s)
  • Function Overloading
    Function Overloading is a feature in C++ that allows functions to possess the same name but contain different parameters.
  • Default Arguments
    Default Arguments are values that can be assigned within the declaration of a function. Default arguments are automatically assigned if the user fails to provide a value for a specified input.
  • Storage Classes
    Storage Classes in C++ specify the scope, lifetime, and visability of functions, which determine where and how functions can be accessed. The storage classes that can be used in C++ include: auto, extern, static, register, and mutable.

Object-Oriented Programming (OOP)

  • Classes and Objects
    Classes and Objects are features of C++ that allow for user-defined data types. Users can define classes, which contain a set of data members and member functions. Classes allow for the creation of objects.
  • Inheritance
    Inheritance is a principle of OOP that allows a child class to automatically inherit all properties and behaviors of its parent class.
  • Encapsulation
    Encapsulation restricts access to an object's components, protecting data integrity and privacy. Members can be declared as private, protected, or public. Private members are only accessible within the class itself. Protected members are accessible within the class and other derived classes. Lastly, Public members are accesible outside the class.
  • Polymorphism
    Polymorphism is a key feature of OOP that allows a single function to behave differently based on the type of the input object, enhancing code flexility.
  • Templates
    Templates allow a user to write a single function definition that works with multiple data types.

Data Structures

  • Vector
    A Vector is a dynamic array, meaning the size can change dynamically. Vectors are part of the C++ Standard Template Library (STL).
  • Stack
    Stacks are also a member of the STL library. It is a container that supports Last-in-First-Out (LIFO) ordering.
  • List
    Lists in C++ are STL containers with randomized storage that contain pointers to the previous and next elements in the list. They are otherwise known as Doubly Linked Lists.
  • Queue
    Queues are data containers that support First-in-First-out ordering in which the first element inserted will consequentially be the first removed.
  • Priority Queue
    Priority Queues differ from regular queues in that data is removed based on its priority instead of the order it was added.
  • Deque
    A Deque is a double-ended queue that allows for insertion and deletion at both ends.
  • Array
    Arrays in C++ utilize a fixed block of memory to store data of the same type.
  • Map
    Maps are containers that utilize key-value pairs and allows for quick and efficient data lookup.
  • Multimap
    A Multimap is a map that allows multiple elements to possess the same key.
  • Unordered Map
    An Unordered Map is a map that does not keep key-value pairs in a sorted order. Instead, they are implemented using a hash table.
  • Set
    Sets are containers in C++ that store unique elements usually in ascending order. They differ from a list in that no duplicate elements are allowed.
  • Unordered Set
    Unordered Sets differ from sets in that they don't store elements in any specific order.

Design Patterns in C++

  • Observer Pattern
    The Observer Pattern in C++ is a behavioral design pattern that allows objects to be notified by other objects about a change in their state. Specifically, it outlines a dependency between objects.
  • Command Pattern
    The Command Pattern's primary focus is the conversion of a request to that of an object, allowing for the flexible queueing of commands as well as undo/redo functionality.
  • Singleton Pattern
    The Singleton Pattern asserts that only one instance of a class can exist at a single time and is the simplest of all design patterns.
  • Builder Pattern
    The Builder Pattern separates the construction of an object class from its representation, allowing the same construction protocol to produce different types.
  • Factory Pattern
    The Factory Pattern involves the creation of an interface or "factory" for creating objects, which allows its subclasses to alter the type of objects created.
  • Abstract Factory Pattern
    Similar to the factory design pattern, the Abstract Factory Pattern involves the creation of a factory interface for creating groups of related objects.
  • Null Object Pattern
    The Null Object Pattern works by defining default behavior for the absence of an object, or a "Null Object", only when necessary.
  • Strategy Pattern
    The Strategy Pattern defines a set of algorithms or "strategies" that contain variations in behavior implemented as distinct classes.
  • Adapter Pattern
    The Adapter Pattern is a design pattern that involves an adapter class, which converts an existing interface to that of another class, adapting it to the instance of a different object.
  • State Pattern
    The State Pattern is a behavioral design pattern that allows an object to change its behavior based on a change in internal state.
  • Design Pattern Classification
    Design patterns can be classified into three main groups: Creational, Structural, and Behavioral. Creational Patterns introduce object creation methods for modifying the instantiation process in a way suitable for the program. Structural Patterns involve the establishment of relationships between classes. Behavioral Patterns define how objects should behave and communicate with other objects.

Memory Management

  • New
    The New Operator in C++ allows a user to request for memory allocation. If sufficient memory exists, the memory will be reserved and the address of the newly allocated memory is returned.
  • Malloc
    Malloc is used to allocate a block of memory specified by a size of bytes, returning a pointer to the first byte of allocated memory.
  • Calloc
    Calloc requests the allocation of a block of memory for an array of objects, each of a specified length size of bytes, and initializes all of the memory to zero.
  • Delete
    The Delete Operator is used to deallocate memory previous allocated by the new operator, allowing the memory to be reused.
  • Malloc vs Calloc vs Realloc
    Malloc vs Calloc vs Realloc are all functions in C++ involving memory allocation, but they serve different purposes. Malloc allocates a block of memory specified in bytes. Calloc allocates memory for an array of elements. Realloc changes the size of a previously allocated memory block.
  • Static Memory Management
    Static Memory Allocation involves automatic memory allocation at compile time of a program and deallocation by the runtime environment once the program finishes running.
  • Dynamic Memory Management
    Dynamic Memory Allocation occurs when a user manually allocates using new or deallocates memory using delete or free.
  • Memory Leaks in C++: How to Prevent Them
    Memory Leaks occur when a program fails to deallocate memory after it is no longer needed, leading to the gradual accumulation of used resources. They can be avoided by always releasing dynamically allocated memory once it is done being used.
